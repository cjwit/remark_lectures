Proposed system * Using arbitrary and compact polygons facilitates readability * Boundaries can better show relationships * Approximations and sampling alleviate computational issues * Color, transparency, and text labels can add metric information .center.image-40[![org/jfree]({{site.baseurl}}/assets/jfree.png)] --- # Process * Automatic hierarchy extraction from OO source code * Computation of Voronoi tessellations * Iterative and potentially costly process * Numeric instead of an analytical approach * Build polygons from curves * Find curves between adjacent circles * Select the curves relevant to the polygons * Clip them to the polygons' borders to create shapes --- # Additional user-centric adaptations * Allowing user interaction * Interactivity through zooming * Ability to explore data .center.image-50[![Figure 10]({{site.baseurl}}/assets/fig10.png)] --- # Additional user-centric adaptations * Information hiding and scoping * Using increasing transparency to visually render hierarchy * Lower levels are less opaque or hidden * Ability to animate according or adjust based on a transparency threshold .center.image-40[![Figure 11]({{site.baseurl}}/assets/fig11.png)] --- #
